DATABASE

Q.1 What is Database?

A database is a structured and organized collection of data that is stored electronically. It is designed to efficiently store, retrieve, and manage information. Databases are used in various applications and industries to store, organize, and manipulate data for purposes such as data analysis, reporting, and decision-making.

Q.2 Types of Databases

1. Relational Database :
In this, the data is stored in a table of rows and columns
2. Object Oriented Databases :
In this, the data is stored in form of objects.
3. NoSQL database :
It allows to store the data which is unstructured or semistructured.
4. Graph database :
The data gets stored in the form of entities and the relationship between them.

Q.5 What is MongoDB?

  • MongoDB is a popular open-source document-oriented NoSQL database management system that is designed to store and manage unstructured or semi-structured data.
  • It works on the concept of Collections and documents.
  • Collections: A collection is a group of MongoDB documents. We can have multiple collections in a single database. You can consider these collections are like tables that we have in relational databases
  • Documents: A document is a set of key-value pairs. These documents can have dynamic schema which means that data present in same document need not to have the same set of fields or structure just because they belong to same collection.
  • Value can be any datatype. Some of them are String, Number, Boolean, Arrays, Timestamp, Object, Null, Date, Object ID

Q.6 In what way mongoDb is better than MySQL??

Compared to MySQL, MongoDB is easier to work with. Since there is no need to define schemas.As, MongoDB can store large amounts of unstructured data much faster than MySQL. It follows a document-based storage approach means MongoDB stores data in a single document for an entity and helps in faster data read or write.

Q.9 How to truncate primary key foriegn key in mongoDb?

In MongoDB, you don't have the concept of "truncate" for primary keys (referred to as "_id" by default) or foreign keys like you do in traditional relational databases.

1. Remove All Documents from a Collection:
db.collectionName.deleteMany({});
2. Remove Documents Based on a Filter:
db.collectionName.deleteMany({ field: value });
3. Delete the entire collection:
db.collectionName.drop();

Q.10 What is Mongoose ? Why do we use it?

Mongoose is an Object Data Modeling (ODM) library for Node.js and MongoDB. It simplifies interaction with MongoDB databases by providing a schema-based abstraction.

We use it bcz it enables Schema Definiton, provides built-in validation and intregate with Promises and async & await for asynchronous operation.

Q.11 What is RDBMS?

RDBMS stands for Relational Database Management System. It organizes data into row and columns which collectively form a table.

What is Schema?

A schema in the context of databases is a blueprint or structure that defines how data is organized within a database system
